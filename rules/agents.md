# Agent 编排

## 可用 Agent

位于 `~/.claude/agents/`:

| Agent | 用途 | 使用场景 |
|-------|---------|-------------|
| quest-designer | 闯关大纲设计 v4 | 复杂功能、重构、完整代码蓝图 |
| architect | 系统设计 | 架构决策 |
| tdd-guide | 测试驱动开发 | 新功能、Bug 修复 |
| code-reviewer | 代码审查 | 编写代码后 |
| security-reviewer | 安全分析 | 提交前 |
| build-error-resolver | 修复构建错误 | 构建失败时 |
| e2e-runner | E2E 测试 | 关键用户流程 |
| refactor-cleaner | 死代码清理 | 代码维护 |
| doc-updater | 文档更新 | 更新文档 |

## 立即使用 Agent

无需用户提示即可使用：
1. 复杂功能请求 - 使用 **quest-designer** agent
2. 刚编写/修改的代码 - 使用 **code-reviewer** agent
3. Bug 修复或新功能 - 使用 **tdd-guide** agent
4. 架构决策 - 使用 **architect** agent

## 并行任务执行

对于独立操作，始终使用并行 Task 执行：

```markdown
# 正确: 并行执行
并行启动 3 个 agent：
1. Agent 1: auth.ts 安全分析
2. Agent 2: 缓存系统性能审查
3. Agent 3: utils.ts 类型检查

# 错误: 不必要的顺序执行
先执行 agent 1，然后 agent 2，最后 agent 3
```

## 多视角分析

对于复杂问题，使用分角色子 agent：
- 事实审查员
- 高级工程师
- 安全专家
- 一致性审查员
- 冗余检查员

## Agent 循环模式

每个 Agent 内部遵循 **感知 -> 思考 -> 行动 -> 验证** 的四步循环：

### 循环步骤

```
1. 感知 (Perceive)
   - 读取项目目录结构、文件内容
   - 查看 Git 历史、最近变更
   - 收集错误信息、测试结果
   ↓
2. 思考 (Think)
   - 分析跨文件依赖关系
   - 规划最佳实现方案
   - 评估风险和影响范围
   - 选择合适的工具和策略
   ↓
3. 行动 (Act)
   - 编辑文件、创建新文件
   - 运行命令（构建、测试、部署）
   - 执行 Git 操作
   ↓
4. 验证 (Verify)
   - 检查行动结果是否符合预期
   - 运行测试确认无回归
   - 根据结果自动修正
   - 如果未完成，回到步骤 1
```

### 与 Quest Designer v4 的关系

| Agent 循环 | Quest Designer 阶段 | 产出 |
|-----------|---------------------|------|
| 感知 | PHASE 1: DISCOVER | 项目上下文 + 能力清单 |
| 思考 | PHASE 2: REASON | Quest Map（完整代码蓝图） |
| 行动 | PHASE 3: EXECUTE | 逐关执行，增量提交 |
| 验证 | PHASE 4: VERIFY | 全量门禁（编译 + 测试 + 安全） |

### 与 Canonical Router 的关系

Canonical Router 在 **思考** 阶段发挥作用：
1. 接收用户意图（感知阶段的输出）
2. 分析意图关键词 + 评估复杂度 + 检测安全敏感度（思考阶段的核心逻辑）
3. 推荐最合适的 Agent + 回退链（行动阶段的决策依据）
4. 路由结果由 PHASE 4 验证（验证 Agent 选择是否正确）

### 最佳实践

1. **单次循环应有明确目标** -- 每个 Agent 调用应该完成一个明确的任务，不要在一个循环中混合多个不相关的目标

2. **验证失败时缩小范围** -- 如果验证步骤发现错误，不要重新执行整个循环，而是聚焦于错误点进行局部修复

3. **感知阶段收集足够信息** -- 不要在信息不足时就开始行动。先读文件、理解上下文、检查依赖，再动手编码

4. **行动阶段遵守蓝图** -- 如果有 Quest Map，严格按照蓝图代码执行。不要"优化"或"改进"蓝图中的代码，除非蓝图本身有错误

5. **验证阶段用命令而非主观判断** -- 用 `npm test`、`mvn compile`、`grep` 等命令验证，不要靠"看起来对"来判断

### 循环终止条件

Agent 循环在以下条件之一满足时终止：
- 所有验收标准通过（绿灯）
- 达到最大重试次数（默认 2 次）
- 用户手动中断
- 遇到不可恢复的错误（需要人工介入）
